# Marketing Copy

## Short Description

An Excel-based AI Security & Governance Checklist for organisations adopting ChatGPT, Claude, Copilot, AI coding assistants, LLM applications, RAG, and agentic AI workflows.

## Problem Statement

AI adoption is moving faster than enterprise security controls. Employees are uploading documents to AI tools, developers are using AI coding assistants, teams are experimenting with RAG and agents, and security teams often lack visibility, guardrails, and detection coverage.

This tool gives organisations a practical way to assess and improve AI security maturity.

## LinkedIn Launch Post

I built an **Enterprise AI Security & Governance Checklist** for organisations adopting GenAI, LLMs, copilots, AI coding assistants, RAG, and agentic AI workflows.

The problem is simple: AI adoption is moving faster than security governance.

Employees are using ChatGPT, Claude, Gemini, Copilot, AI browser extensions, document summarisation tools, and AI coding assistants every day. That creates real risks around sensitive data leakage, Shadow AI, prompt injection, risky OAuth apps, insecure AI-generated code, and agentic workflows that can take actions across internal systems.

This checklist helps organisations assess controls across:

- AI asset inventory
- Approved AI tool governance
- Data classification
- DLP and secrets detection
- LLM gateway controls
- Prompt and output guardrails
- Shadow AI detection
- AI vendor review
- AI incident response
- AI security maturity scoring

It is mapped to ideas from NIST AI RMF, ISO/IEC 42001, OWASP LLM Top 10, MITRE ATLAS, and other recognised AI security and governance resources.

The goal is not to block AI adoption. The goal is to help organisations adopt AI safely.
